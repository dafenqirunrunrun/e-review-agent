# Public Business E2E

The public E2E test drives real HTTP APIs against the Compose runtime.

## Scenarios

1. Verify public AI mode:
   - `GET /health/ready`
   - Assert `runtimeMode=public-rule`, `engineType=rule`, `modelLoaded=false`.

2. Submit a normal product review:
   - Login customer through `/wx/auth/login`.
   - Submit a real review through `/wx/order/comment`.
   - The comment is stored in `litemall_comment`.

3. Submit a high-risk product review:
   - Use a seeded reviewable order.
   - Submit a low-score review with refund and safety terms.
   - The comment is stored in `litemall_comment`.

4. Trigger Agent patrol:
   - Login admin through `/admin/auth/login`.
   - Call `/admin/ai/patrol/run-once`.
   - Verify analysis rows from `sourceType=litemall_comment`.

5. Verify risk task generation:
   - Query `/admin/ai/risk/list`.
   - Ensure at least one pending risk task is generated from a real comment analysis.

6. Verify idempotency:
   - Trigger patrol a second time.
   - Verify analysis and risk counts do not increase for already analyzed comments.

7. Verify operation handling:
   - Call `/admin/ai/operation/handle`.
   - Verify dashboard/summary changes after processing.

8. Verify AI unavailable behavior:
   - Stop `ai-service`.
   - Submit a comment and trigger patrol.
   - Verify analysis is not falsely marked successful.
   - Restart `ai-service` and verify recovery.

## Important Boundary

Prerequisite order state is prepared through public seed data. Comment analysis and risk-task results are generated by the real application workflow.

The E2E test must not directly insert rows into:

- `litemall_review_ai_analysis`
- `litemall_ai_review_risk_task`
- `litemall_ai_operation_log`
